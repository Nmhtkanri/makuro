# 経費一覧表マクロ — README

## 概要

jinjer・e-staffing・Freee の 3つの経費管理システムからデータを取り込み、  
社内の「集計」シートへ一元集計したうえで、CSV出力やカラー勤務表への書き込みを行うマクロ群です。

---

## 処理フロー

```
┌─────────── 入力層 ───────────┐
│  jinjer CSV                  │
│  e-staffing CSV/Excel        │──→「経費一覧表」シートに統合
│  Freee 本決算経費            │      ↓
└──────────────────────────────┘   社員ID照合 → 重複削除
                                      ↓
┌─────────── 集計層 ───────────┐
│  c設定.bas                   │──→ キーワード照合・カテゴリ集計
│  「設定」シートのキーワード   │      ↓
└──────────────────────────────┘  「集計」シート生成 (C～K列)
                                      ↓
┌─────────── 振り分け層 ───────┐
│  dRY列振り分けマクロ         │──→ C～K列 → R～Y列へ振り分け
│  I追加金額マクロ             │──→ Q列 → X/Y列にマージ
│  仕訳データ振り分けマクロ    │──→ 通信手当等 → R/S, T/U列に配分
└──────────────────────────────┘
                                      ↓
┌─────────── 出力層 ───────────┐
│  Module2 (CSV出力)           │──→ jinjer用 CSVファイル
│  Fカラー勤務表へ書き込み     │──→ 各社員のカラー勤務表ファイル
└──────────────────────────────┘
```

## モジュール一覧

### メイン制御

| ファイル | 主要プロシージャ | 役割 |
|---------|----------------|------|
| **a一連の処理.bas** | `Append_全て_統合処理()` | メインオーケストレーター。入力層の各取込を順番に呼び出す |

### 入力層（データ取込・整形）

| ファイル | 主要プロシージャ | 役割 |
|---------|----------------|------|
| **E一覧表作成マクロ.bas** | `Append_経費統合一覧_to_経費一覧表()` | jinjer+Freee統合データを「経費一覧表」へ追記 |
| | `Append_e_staffing_経費_to_経費一覧表()` | e-staffingデータを取込 |
| | `Append_jinjer_CSV_to_経費一覧表()` | jinjer CSVを直接取込 |
| | `Append_Freee_to_経費一覧表()` | Freeeデータを取込 |
| | `AssignEmployeeNo_ByName_じんじてJinjer()` | 名前から社員番号を照合・付与 |
| | `RemoveDuplicates_A_D_F_AndLog()` | A/D/F列ベースで重複行を削除 |
| **b経費統合一覧表の取り込みマクロ.bas** | `Append_本決算経費_to_経費一覧表()` | Freee本決算経費シートからの取込 |
| **jinjer経費貼り付けマクロ.bas** | — | jinjer CSVをシートに貼り付け |
| **estaffing貼り付けマクロ.bas** | — | e-staffing CSV/Excelをシートに貼り付け |
| **estaffing整形マクロ.bas** | `Export_EStaffing_SelectedColumns()` | e-staffingデータの列整形 |
| **GFREE整形マクロ.bas** | `Freee〜削除〜名前の変換〜最終済()` | Freeeデータの整形（ヘッダー保持、列削減、重複排除） |

### 集計層

| ファイル | 主要プロシージャ | 役割 |
|---------|----------------|------|
| **c設定.bas** | `Run_経費集計_設定シート対応()` | 「設定」シートのキーワードで経費を分類し「集計」シートへ出力 |
| | `Setup_設定シート作成()` | 「設定」シートの初期作成 |

### 振り分け層

| ファイル | 主要プロシージャ | 役割 |
|---------|----------------|------|
| **dRY列振り分けマクロ.bas** | `Distribute_To_RY_Columns()` | 集計シート C～K列 → R～Y列に振り分け |
| **I追加金額マクロ.bas** | `Merge_Q_To_XY()` | Q列の値をX/Y列にマージ |
| **仕訳データ振り分けマクロ.bas** | `仕訳データ蓖振分け_RS_TU()` | 仕訳データの通信手当等をR/S, T/U列に配分 |

### 出力層

| ファイル | 主要プロシージャ | 役割 |
|---------|----------------|------|
| **Module2.bas** | `経費一覧CSV作成()` | 集計シートをjinjer用CSV形式に変換・出力 |
| **Fカラー勤務表へ書き込み.bas** | `ProcessPathList()` | PathListシートを参照し、各社員のカラー勤務表ファイルにR～Y列の値を書き込む |

### ユーティリティ

| ファイル | 主要プロシージャ | 役割 |
|---------|----------------|------|
| **Aパスリストから開くマクロ.bas** | `OpenReport_CurrentRow()` / `OpenFolder_CurrentRow()` | パスリストのC列パスからファイルやフォルダを開く |
| **Module1.bas** | — | 旧バージョン・廃止マクロ（参考保存） |

## 集計シートの列構成

### C～L列（集計結果）

| 列 | 内容 |
|---|------|
| A | 社員番号 |
| B | 名前 |
| C | 合計 (F+G+H+I+J+K) |
| D | 顧客勤務手当 |
| E | RINK手当 |
| F | 手当2 (D+E) |
| G | 顧客請求額 |
| H | 交通費 |
| I | 非課税立替(郵便局) |
| J | その他(会議費・消耗品など) |
| K | テレワーク手当 |
| L | 計上日 |

### R～Y列（振り分け先）

| 列 | 内容 | 振り分け元 |
|---|------|-----------|
| R | 手当1（名称） | F列 → "顧客勤務手当" |
| S | 手当1（金額） | F列の値 |
| T | 手当2（名称） | — |
| U | 手当2（金額） | — |
| V | 通勤交通費 | H列 |
| W | 顧客請求(立替金) | G列 |
| X | 非課税立替 | I列 |
| Y | その他 | J列 |

## 基本的な使い方

### 一括実行（通常フロー）

1. jinjer / e-staffing / Freee の各データをシートに貼り付ける
2. **`Append_全て_統合処理()`** を実行（a一連の処理.bas）
3. **`Run_経費集計_設定シート対応()`** を実行（c設定.bas）
4. **`Distribute_To_RY_Columns()`** を実行（dRY列振り分けマクロ.bas）
5. 必要に応じて `仕訳データ蓖振分け_RS_TU()` / `Merge_Q_To_XY()` を実行
6. **`経費一覧CSV作成()`** でCSV出力、または **`ProcessPathList()`** で勤務表書込

### 個別実行

各モジュールは独立して実行可能です。  
例えば `Distribute_To_RY_Columns()` だけを再実行して R～Y列を更新することもできます。

## 注意事項

- **エンコーディング:** すべての .bas ファイルは Shift-JIS (cp932) で保存されています。UTF-8 で保存するとVBEでインポート時に文字化けします。
- **「設定」シート:** c設定.bas のキーワード照合ロジックに直結しています。カテゴリの追加・変更はこのシートで管理してください。
- **バックアップ:** 集計処理実行時にバックアップシート（例: `集計_backup_20260408_123456`）が自動作成されます。
